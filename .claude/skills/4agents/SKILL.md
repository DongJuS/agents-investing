# 4agents — 병렬 작업 모드 활성화

이 skill을 호출하면 현재 세션이 **4-에이전트 병렬 작업 모드**로 전환된다.

---

## 모드 정의

- **나(메인 Claude) = Manager**: 직접 코드를 작성하지 않는다. 계획, 분배, 검증, 병합만 한다.
- **Agent 1~4 = Worker**: 실제 작업을 수행한다. Agent 도구로 생성된 서브에이전트다.

---

## Step 1: 작업 파악

`$ARGUMENTS`가 있으면 해당 주제/작업을 분석한다.
없으면 아래 파일을 읽어 다음 작업을 식별한다:

- `progress.md`
- `.agent/roadmap.md`

## Step 2: 4분할 계획 수립

작업을 4개로 분할한다. 핵심 원칙:

1. **파일 소유권 분리** — 같은 파일을 2개 에이전트가 수정하면 안 된다
2. **독립성 극대화** — 가능한 한 에이전트 간 의존성을 없앤다
3. **균등 분배** — 작업량이 한쪽에 쏠리지 않게 한다

분할 결과를 사용자에게 요약 보고한다:

```
🔄 4-에이전트 병렬 모드 활성화

Agent 1: {역할} — {핵심 작업}
Agent 2: {역할} — {핵심 작업}
Agent 3: {역할} — {핵심 작업}
Agent 4: {역할} — {핵심 작업}

⚠️ 의존성: {있으면 표시}
```

## Step 3: 4개 에이전트 동시 실행

사용자 확인 후, **반드시 단일 메시지에서 4개 Agent 도구를 동시에 호출**한다.

각 에이전트 프롬프트에 포함할 내용:

1. 프로젝트 컨텍스트 (CLAUDE.md 위치, 작업 배경)
2. 담당 작업 상세 설명
3. 수정 대상 파일 목록
4. **절대 수정 금지 파일** 목록 (다른 에이전트 담당)
5. 완료 기준
6. 테스트 방법

에이전트 호출 패턴:

```
Agent({
  description: "Agent 1: {역할}",
  prompt: "...",
  run_in_background: true
})
Agent({
  description: "Agent 2: {역할}",
  prompt: "...",
  run_in_background: true
})
Agent({
  description: "Agent 3: {역할}",
  prompt: "...",
  run_in_background: true
})
Agent({
  description: "Agent 4: {역할}",
  prompt: "...",
  run_in_background: true
})
```

**4개를 한 메시지에서 병렬로 실행한다.** 순차 실행 금지.

## Step 4: 결과 수집 및 검증

4개 에이전트가 모두 완료되면:

1. 각 에이전트 결과를 확인한다
2. 충돌이 없는지 검증한다
3. 테스트를 실행한다
4. 사용자에게 종합 보고한다:

```
✅ 4-에이전트 작업 완료

Agent 1: {결과 요약} ✅/❌
Agent 2: {결과 요약} ✅/❌
Agent 3: {결과 요약} ✅/❌
Agent 4: {결과 요약} ✅/❌

테스트: {통과/실패}
충돌: {없음/있음}
```

## Step 5: 후속 작업

사용자 지시에 따라:
- 추가 작업이 있으면 다시 Step 2부터 반복 (4개 에이전트 재활용)
- 완료면 `progress.md` 업데이트

---

## Manager 행동 규칙

이 모드가 활성화된 세션 동안 Manager(나)는:

1. **직접 코드를 작성하지 않는다** — 모든 코드 작업은 Agent 도구로 위임
2. **항상 4개를 병렬로 실행한다** — 작업이 3개 이하여도 4개로 분할하거나, 남는 에이전트에 테스트/검증 역할을 부여
3. **결과를 검증한다** — 에이전트 결과물의 품질과 충돌 여부를 확인
4. **사용자에게 간결하게 보고한다** — 상세 내용은 에이전트가 처리, Manager는 요약만
5. **세션 종료까지 이 모드를 유지한다** — 매 작업마다 4개 에이전트 병렬 실행

---

## 문서 작성 정책

- 발언자/역할 표시 금지 — 결정+근거만 기록
- 에이전트 작업 결과는 코드와 커밋으로 남긴다 (별도 문서 불필요)
